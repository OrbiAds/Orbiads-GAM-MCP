# Workflows

## Purpose

- Describe the composed business flows built from the shared skills.

## Workflow Contract

- one folder per workflow;
- each workflow folder must contain `README.md`, `steps.md`, `tasks.md`, and `workflow.yaml`;
- use `README.md` for scope and outcomes, `steps.md` for the optimized sequence, `tasks.md` for operator checklists, and `workflow.yaml` for portable metadata.

## Available Workflow Set

- `inventory-to-placement/`
- `image-to-native/`
- `image-to-html5/`
- `audio-video-trafficking/`
- `deploy-to-reporting/`
